# E2 本地验证说明

## 1. 适用范围

本文用于验证 Epic 2「文档中心」的 Sprint 03 最小链路：

- 文档、文档版本、`IngestJob` 和 `stored_files` 迁移。
- 文档上传元数据接口。
- 文档列表、详情、版本列表、作业列表和作业详情接口。
- 前端 P06 文档中心、P07 文档详情真实接口接入。

本文不覆盖真实 MinIO 写入、解析 Worker、Chunk 生成、检索副本、重解析、active version 切换和完整 ACL。

## 4. 后端接口验证

编译检查：

```powershell
conda run -n rag-lab python -m compileall app migrations
```

上传文档并验证返回文档、版本和 queued 作业：

```powershell
conda run -n rag-lab python -c "from fastapi.testclient import TestClient; from app.main import app; c=TestClient(app); kb=c.get('/api/v1/knowledge-bases').json()['items'][0]['kbId']; r=c.post(f'/api/v1/knowledge-bases/{kb}/documents', files={'file': ('e2-smoke.txt', b'E2 smoke document', 'text/plain')}, data={'name':'E2 Smoke Document','securityLevel':'public'}); print(r.status_code); print(r.json())"
```

查询文档、详情、版本和作业：

```powershell
conda run -n rag-lab python -c "from fastapi.testclient import TestClient; from app.main import app; c=TestClient(app); kb=c.get('/api/v1/knowledge-bases').json()['items'][0]['kbId']; docs=c.get(f'/api/v1/knowledge-bases/{kb}/documents').json(); doc=docs['items'][0]['documentId']; print(docs); print(c.get(f'/api/v1/knowledge-bases/{kb}/documents/{doc}').json()); print(c.get(f'/api/v1/knowledge-bases/{kb}/documents/{doc}/versions').json()); print(c.get(f'/api/v1/knowledge-bases/{kb}/ingest-jobs?documentId={doc}').json())"
```

验证通过标准：

- 上传接口返回 `201`。
- 响应中同时包含 `document`、`version`、`ingestJob` 和 `storedFile`。
- 新建 `ingestJob.status` 为 `queued`。
- 文档列表能看到新上传文档。
- 文档详情包含 `activeVersion`。
- 版本列表至少包含 `v1`。
- 作业列表能按 `documentId` 返回关联作业。

## 6. 当前边界

- 开发期只保存文件元数据、checksum 和对象引用，不写入真实 MinIO。
- 上传接口在同一事务中创建 `stored_files`、`documents`、`document_versions` 和 `ingest_jobs`。
- `DocumentVersion` 初始状态为 `processing`，`parseStatus` 为 `pending`，作业状态为 `queued`。
- 完整权限矩阵、Chunk 查询、重解析、active version 切换和 Worker 状态推进留到后续 Sprint。
