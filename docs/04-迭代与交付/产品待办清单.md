# 产品待办清单

## 1. 使用说明

- 用于承接需求条目、技术任务、缺陷和优化项。
- 推荐按优先级和版本目标管理。
- 每条待办建议能映射到需求、原型和设计文档。
- 本清单是长期 backlog，不等同于当前 Sprint 计划。
- 每个 Sprint 开始前，从状态为 Ready 或高优先级 Todo 的条目中选择本轮交付范围，写入对应迭代计划。

## 2. 版本目标

V1.0 的开发目标是先完成“可运行、可配置、可追溯”的 RAG 调试平台最小闭环：

1. 本地工程可启动，前后端基础链路可验证。
2. 用户、知识库、文档、配置和 QA Run 有稳定数据模型与接口。
3. 文档入库、配置激活、QA 调试和历史查看可以形成最小闭环。
4. 真实 LLM / Embedding / 检索组件通过 Provider 抽象逐步替换 mock 能力。

## 6. Sprint 取项规则

- Sprint 计划只选择本轮要交付和验证的条目，不复制完整 backlog。
- P0 且状态为 Ready 的条目优先进入 Sprint。
- 如果 Sprint 中发现待办范围过大，应拆分为更小条目后再推进。
- Sprint 结束后，将完成状态回写到本清单，并补充新增缺陷或后续任务。
