# 迭代计划 Sprint 02

## 1. Sprint 基本信息

- Sprint 名称：Sprint 02
- Sprint 主题：E1 用户与知识库基础
- 时间范围：待定
- 目标：完成开发期认证占位、用户/用户组基础数据结构、知识库基础接口，并让前端平台工作台可以读取真实知识库数据。

## 2. 本 Sprint 目标

- 建立开发期认证占位，保证后续受控接口有统一当前用户上下文。
- 建立用户、用户组、知识库相关最小数据模型与迁移基础。
- 实现知识库列表、详情和创建接口。
- 前端平台工作台和知识库概览入口接入真实接口。
- 保持范围克制：不实现完整权限矩阵、真实登录密码体系、文档中心、QA Run 或外部检索组件。

## 4. 建议实现顺序

1. 后端认证占位：提供稳定当前用户上下文，先不做真实密码登录。
2. 数据库迁移基础：引入迁移工具，创建用户、用户组、知识库表。
3. 种子数据：提供本地开发默认用户和默认知识库。
4. 知识库 API：实现列表、详情、创建接口，返回接口设计说明中的 camelCase DTO。
5. 前端接入：新增 service / adapter / type 分层，P02/P05 优先读取真实知识库数据。
6. 验证与文档：补充本地初始化、迁移、启动和接口验证命令。

## 5. 验收标准

- 后端可以在 `rag-lab` Conda 环境下完成迁移或初始化。
- 本地至少有一个开发期用户和一个可见知识库。
- `GET /api/v1/auth/me` 能返回当前开发用户摘要。
- `GET /api/v1/knowledge-bases` 能返回当前用户可见知识库列表。
- `GET /api/v1/knowledge-bases/{kbId}` 能返回知识库详情。
- `POST /api/v1/knowledge-bases` 能创建知识库，并返回新建对象。
- 前端平台工作台不再只依赖静态 mock 数据，能读取知识库列表接口。
- 前端构建、后端编译和核心接口 TestClient 验证通过。

## 6. 范围边界

- 不实现生产级登录、密码、JWT 刷新、SSO 或权限后台管理。
- 不实现完整 ACL、deny 优先细粒度权限和 Chunk 级权限。
- 不接入 PostgreSQL 以外的外部组件。
- 不实现文档上传、配置中心、QA 调试或图检索。
- 不重构原型视觉和页面布局，只做接入真实数据所需的最小改动。

## 7. 已确认信息

- 开发期内置两个用户：`admin` 和 `user`。
- 默认当前用户使用 `admin`。
- Sprint 02 本地数据库直接使用 PostgreSQL，连接信息后续写入 `.env`。
- 默认开发用户可创建知识库。
- 默认密级保持开放，使用 `public`，避免影响开发联调。

## 8. 风险与阻塞

- 如果认证方案过早做成生产级，会拖慢 E1 主线。
- 如果数据库迁移工具和 ORM 选择不明确，会影响后续所有模块。
- 如果前端直接绑定后端 DTO，后续接口演进会增加返工。

## 9. Sprint 输出物

- 知识库列表、详情、创建接口。
- 前端平台工作台真实接口接入。
- E1 本地验证说明。

## 10. 决策记录

### S2-001 开发期认证占位

- 决策：内置 `admin` 和 `user` 两个开发用户，默认当前用户为 `admin`。
- 范围：只提供 `GET /api/v1/auth/me` 和后端依赖 `get_current_user`，不实现真实登录、密码、JWT 或 SSO。
- 已完成：新增开发期用户服务、认证依赖、当前用户接口和配置项。
- 后续：S2-003 知识库接口复用 `get_current_user` 判断创建和可见范围。

### S2-002 用户、用户组基础表与迁移

- 决策：引入 Alembic 作为数据库迁移工具，连接串统一从 `RAG_LAB_DATABASE_URL` 或 `DATABASE_URL` 读取。
- 范围：创建 `users`、`user_groups`、`user_group_members` 三张基础表和必要索引约束，不实现完整权限矩阵、ACL 或知识库表。
- 已完成：新增首个迁移版本、迁移执行脚本和后端迁移说明；已在开发 PostgreSQL 执行到 `head`。
- 后续：S2-003 复用迁移基础新增知识库表、开发期种子数据和知识库接口。

### S2-003 知识库基础表与接口

- 决策：E1 阶段采用最小可见性规则，平台管理员可见全部知识库，普通用户可见自己负责的知识库；完整成员绑定和权限矩阵留到后续迭代。
- 范围：新增 `knowledge_bases` 表、开发期 `admin/user` 和默认知识库种子数据，提供知识库列表、详情和创建接口。
- 已完成：`GET /api/v1/knowledge-bases`、`GET /api/v1/knowledge-bases/{kbId}`、`POST /api/v1/knowledge-bases` 可用；`GET /api/v1/auth/me` 返回真实可见知识库数量。
- 环境处理：开发库中存在旧版 `knowledge_bases` 表时，迁移会保留旧数据并重命名为 `legacy_knowledge_bases`，再创建符合当前设计的新表。
- 后续：S2-004 前端平台工作台接入真实知识库列表和详情入口。
