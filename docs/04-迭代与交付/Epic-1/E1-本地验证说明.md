# E1 本地验证说明

## 1. 适用范围

本文用于验证 Epic 1「用户与知识库基础」的当前交付结果：

- 开发期认证占位。
- 用户、用户组和知识库基础表迁移。
- 知识库列表、详情、创建、更新和停用接口。
- 前端平台工作台读取真实知识库数据，并支持新建、编辑和停用。

本文不覆盖生产级登录、完整权限矩阵、文档中心、配置中心和 QA Run。

## 2. 环境准备

后端命令均在 `backend/` 目录执行：

```powershell
cd C:\Users\Public\Documents\Code\jin\rag-lab\backend
```

数据库连接优先使用环境变量：

```powershell
$env:DATABASE_URL = (Get-Content .\env_info.txt | Where-Object { $_ -like 'DATABASE_URL=*' } | Select-Object -First 1).Substring('DATABASE_URL='.Length)
```

也可以将连接串写入本地 `backend/.env` 的 `RAG_LAB_DATABASE_URL`。不要提交 `.env` 或包含真实密钥的环境文件。

## 3. 初始化与迁移

安装或更新后端依赖：

```powershell
conda env update -f environment.yml --prune
```

执行数据库迁移：

```powershell
.\scripts\migrate.ps1
```

确认 Alembic 版本：

```powershell
conda run -n rag-lab python -m alembic current
```

预期当前版本为当前仓库最新 head，例如：

```text
0009_create_chunk_access_filters (head)
```

如果本机 `conda run` 输出 Conda 自身用法提示，可临时改用环境内 Python：

```powershell
C:\Users\crrcd\.conda\envs\rag-lab\python.exe -m alembic current
```

## 4. 种子数据

迁移会写入以下开发期种子数据：

| 类型 | 标识 | 说明 |
| --- | --- | --- |
| 用户 | `admin` | 默认开发管理员，UUID 为 `00000000-0000-0000-0000-000000000001` |
| 用户 | `user` | 普通开发用户，UUID 为 `00000000-0000-0000-0000-000000000002` |
| 知识库 | `默认知识库` | E1 联调用默认知识库，负责人为 `admin` |

如果开发库中已存在旧版 `knowledge_bases` 表，迁移会将其重命名为 `legacy_knowledge_bases`，保留旧数据后创建当前设计使用的新表。

## 5. 后端接口验证

编译检查：

```powershell
conda run -n rag-lab python -m compileall app migrations
```

健康检查：

```powershell
conda run -n rag-lab python -c "from fastapi.testclient import TestClient; from app.main import app; r=TestClient(app).get('/api/v1/health'); print(r.status_code); print(r.json())"
```

当前用户和知识库接口：

```powershell
conda run -n rag-lab python -c "from fastapi.testclient import TestClient; from app.main import app; c=TestClient(app); print(c.get('/api/v1/auth/me').json()); print(c.get('/api/v1/knowledge-bases').json())"
```

知识库详情验证：

```powershell
conda run -n rag-lab python -c "from fastapi.testclient import TestClient; from app.main import app; c=TestClient(app); kb=c.get('/api/v1/knowledge-bases').json()['items'][0]; print(c.get('/api/v1/knowledge-bases/' + kb['kbId']).status_code)"
```

知识库创建、更新、停用和停用后写保护验证建议使用一次性开发库，或使用带时间戳的临时名称：

```powershell
conda run -n rag-lab python -c "from datetime import datetime; from fastapi.testclient import TestClient; from app.main import app; c=TestClient(app); name='E1 临时知识库 ' + datetime.now().strftime('%H%M%S'); created=c.post('/api/v1/knowledge-bases', json={'name': name, 'description':'smoke', 'defaultSecurityLevel':'public', 'sparseIndexEnabled': False, 'graphIndexEnabled': False}).json(); kb=created['kbId']; print('created', created['status']); print('patch', c.patch('/api/v1/knowledge-bases/' + kb, json={'description':'updated smoke'}).status_code); print('disable', c.post('/api/v1/knowledge-bases/' + kb + '/disable').json()['status']); print('blocked', c.patch('/api/v1/knowledge-bases/' + kb, json={'description':'blocked'}).status_code)"
```

预期输出包含 `created active`、`patch 200`、`disable disabled`、`blocked 409`。

预期结果：

- 点击知识库卡片进入概览页后，标题和基础配置来自后端接口。

## 7. 当前边界

- `admin` 默认可见全部知识库。
- `user` 当前只可见自己负责或通过成员绑定授权的知识库。
- 停用知识库采用 `status = disabled`，不会物理删除文档、配置或 QA 历史。
- P05 的统计卡片、最近作业和配置明细仍为占位内容，等待 E2/E3/E4 对应接口。
