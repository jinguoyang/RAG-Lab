# Release 总览

## 1. 定位

Release 面向版本发布，汇总跨 Epic、跨 Sprint 的交付范围、验收状态、环境要求和遗留风险。Sprint 负责过程，Release 负责发布判断。

| --- | --- | --- | --- |
| V1.0 | 可运行、可配置、可追溯的 RAG 调试平台最小闭环 | 验收硬化中 | [V1.0-验收与发布范围](./V1.0-验收与发布范围.md) |
| V1.1 | RAG 质量回归、批量评估和结果视图增强 | 已完成 | [V1.1-质量回归规划](./V1.1-质量回归规划.md) |

| V1.3 | 稳定性观测和协作治理增强 | 规划中 | [V1.3-稳定性与协作治理规划](./V1.3-稳定性与协作治理规划.md) |

## 3. Release 退出标准

- 版本范围内 P0 Backlog 全部 Done。
- 对应 Sprint 验证脚本、前端构建和后端编译通过。
- 验收清单和发布运维文档已回填结果。
- 真实外部 Provider 的环境限制和复测项已记录为发布风险或后续任务。
