# V1.1 质量回归规划

## 1. 版本目标

V1.1 聚焦 RAG 质量闭环增强，在 V1.0 可追溯能力基础上补齐批量评估、结果沉淀、前端回归结果视图、配置差异对比和优化草稿。

## 3. 边界

- 不引入复杂异步调度、定时回归或多环境基准平台。
- 不引入自动 LLM-as-judge；先使用规则判定、人工判定字段或可解释的最小比较结果。
- 继续复用现有 QARun 创建、详情和历史链路，不另建一套执行契约。
- 配置优化草稿必须人工复核，不自动激活。

## 4. 验收方向

- EvaluationRun 能关联 ConfigRevision、样本总数、通过数、失败数和错误摘要。
- EvaluationResult 能关联样本、实际 QARun、判定状态、失败原因和核心 metrics。
- P10 能展示回归批次、通过率、失败原因、关联 revision 和跳转历史运行入口。
- 评估结果能导出，并能与 ConfigRevision 差异对比联动。
- 失败样本可以生成配置优化草稿，但不会自动影响 active revision。
