# V1.2 生产化与知识库治理规划

## 1. 版本目标

V1.2 在 V1.1 质量闭环基础上，推进真实 Provider 生产化接入和知识库治理能力，让平台从本地可验证走向测试环境可复测、知识库质量可维护。

## 3. 边界

- 不建设统一密钥管理平台，仍以环境变量或受控配置注入为主。
- 不要求真实 Provider 必须在本地开发环境在线。
- 不建设复杂文档规则引擎或全文人工编辑器。
- 不删除 PostgreSQL 中的 Chunk 真值，治理动作只影响后续检索和展示状态。

## 4. 验收方向

- Provider 诊断能区分未配置、连接失败、连接成功和 local/mock 降级。
- 检索副本可按知识库或文档版本重建，并能记录失败原因。
- P05 能看到知识库健康概览、治理待办和异常文档入口。
- 文档批量治理操作具备权限校验、确认和审计记录。
