# V1.0 验收与发布范围

## 1. 版本目标

V1.0 目标是完成“可运行、可配置、可追溯”的 RAG 调试平台最小闭环：

- 本地工程可启动，前后端基础链路可验证。
- 用户、知识库、文档、配置和 QA Run 有稳定数据模型与接口。
- 文档入库、配置激活、QA 调试、历史查看、权限裁剪和图支撑诊断形成最小闭环。
- 发布前具备 OpenAPI、审计日志、验收脚本和运维说明。

## 4. 验证入口

- 后端编译：`conda run -n rag-lab python -m compileall app`
- Sprint 12 验收：`conda run -n rag-lab python scripts/verify_sprint12_acceptance.py`
- OpenAPI 导出：`conda run -n rag-lab python scripts/export_openapi.py`
- 发布治理验证：`conda run -n rag-lab python scripts/verify_epic10_release_ops.py`
- 前端构建：在 `frontend/` 下执行 `npm run build`

## 5. 已知风险

- 当前验收主要覆盖本地和 local/mock Provider；真实外部 Provider 的网络级连通性、凭据和服务 SLA 需要在测试环境中补充确认。
- V1.0 已具备 EvaluationSample 沉淀能力，但批量回归运行和结果视图归入 V1.1。
