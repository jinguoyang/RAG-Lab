# Sprint 总览

## 1. 定位

Sprint 是时间盒，用于承接一轮可验证交付。Sprint 不隶属于某个 Epic；它从产品待办中选择若干 Backlog，可能覆盖一个或多个 Epic。

## 3. Sprint 文档约定

每个 Sprint 文档至少包含：

- Sprint 基本信息和涉及 Epic。
- 关键假设和范围边界。
- 计划事项，逐项关联 Backlog。
- 验收标准和验证命令。
- Sprint 结束后回填实际结果、遗留问题和下一步建议。
