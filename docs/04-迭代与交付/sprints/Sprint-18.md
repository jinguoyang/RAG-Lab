# 迭代计划 Sprint 18

## 1. Sprint 基本信息

- Sprint 名称：Sprint 18
- Sprint 主题：协作与治理增强
- 涉及 Epic：E17 协作与治理增强
- 建议版本：V1.3
- 时间范围：待定
- 目标：补齐配置发布记录、QA 协作、审计报表、数据保留策略和有效权限解释能力。

## 2. 关键假设

- 本项目仍是内部 RAG 调试平台，不建设复杂审批流。
- 配置发布记录用于复核和回滚确认，不阻塞已有保存/激活主链路。
- QA 协作只做分享、评论、责任人和处理状态，不做工单系统。
- 权限解释工具只解释后端有效权限结果，不把前端展示当最终授权依据。

## 3. 本 Sprint 目标

- P08 增加配置发布记录、变更说明和回滚确认入口。
- P10 支持 QA Run 分享、评论、责任人和处理状态。
- 审计日志支持报表、导出和数据保留策略说明。
- 提供用户有效权限模拟和权限来源解释工具。

## 5. 验收标准

- 配置发布记录能说明变更来源、变更人、变更摘要、关联评估结果和回滚入口。
- QA Run 可以被授权用户查看协作信息，并保留评论和处理状态。
- 审计报表可以按时间、操作者、资源类型和操作类型筛选导出。
- 有效权限解释工具能说明用户通过平台角色、知识库角色、用户组和 ACL 获得或被拒绝的权限来源。

## 6. 范围边界

- 不建设审批流、工单系统或通知中心。
- 不新增多租户隔离体系。
- 不改变后端权限最终判定原则。
