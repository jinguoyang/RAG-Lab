# 迭代计划 Sprint 07

## 1. Sprint 基本信息

- Sprint 名称：Sprint 07
- Sprint 主题：E6 权限与成员治理
- 时间范围：待定
- 目标：围绕成员绑定、权限摘要、ACL 和 Chunk 访问过滤，完成 P12 成员与权限页可联调的最小闭环。

## 2. 关键假设

- Sprint 07 基于 Sprint 06 的 QA Provider 编排继续推进。
- 本轮优先完成 `B-031` 至 `B-034`。
- E1 遗留的 `B-028`、`B-029` 暂不在本 Sprint 内补齐；成员绑定接口先复用开发期种子用户和用户组基础表。
- PostgreSQL 仍是权限、成员和 Chunk 访问过滤摘要的业务真值中心。
- Chunk 真值表和解析 Worker 尚未落地前，ChunkAccessFilter 先形成可生成、可传递的过滤摘要，不承诺完整生产级检索副本同步。

## 3. 本 Sprint 目标

- 实现知识库成员绑定、成员列表、角色修改和移除接口。
- 实现权限码、角色权限绑定、ACL 规则和权限摘要服务。
- 生成 ChunkAccessFilter，并在检索 Provider 调用前接入过滤摘要。
- 接入 P12 成员与权限页的成员绑定、权限摘要和无权限态。
- 保持开发期本地环境可在无外部检索组件时完成最小验证。

## 5. 验收标准

- 成员列表接口可分页返回知识库下的用户和用户组绑定。
- 成员添加接口能校验主体存在、角色合法和重复绑定冲突。
- 成员角色修改接口只更新当前知识库下的有效绑定。
- 成员移除接口采用失效处理，不物理删除历史授权记录。
- 权限摘要与角色权限绑定能解释当前用户在知识库内的关键权限。
- QA 检索 Provider 调用前能收到后端生成的访问过滤摘要。
- P12 页面能展示成员、权限摘要和无权限态，并通过真实接口完成成员维护。
- 后端应用代码可通过 Python 编译检查，相关接口可通过 TestClient 最小验证。

## 6. 范围边界

- 不补齐完整平台用户/用户组管理页面。
- 不实现生产级统一认证或外部身份源同步。
- 不在本 Sprint 中实现复杂审批流或多租户隔离。
- 不让前端自行推导最终授权结果，权限判断以后端为准。
