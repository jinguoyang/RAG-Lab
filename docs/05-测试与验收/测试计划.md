# 测试计划

## 1. 文档目的

说明测试范围、测试策略、测试阶段安排、环境准备和交付标准。

## 2. 测试范围

- 功能测试
- 接口测试
- 权限测试
- 异常流程测试
- 回归测试

## 3. 测试重点

- 文档上传与重解析
- 版本切换与权限脱敏
- 配置 revision 保存与激活
- QA 调试链路与异常反馈
- QA 历史回放与人工标注

## 4. 测试环境

- 本地环境：前端 Node.js/npm，后端 Conda 环境 `rag-lab`。
- 测试环境：独立 PostgreSQL，外部 Provider 可按阶段使用 local/mock 或真实服务。
- 环境配置通过 `backend/.env` 管理，不在文档或代码中保存密钥。

## 5. 进入测试前提

- 后端迁移已执行完成。
- 前端构建可通过。
- OpenAPI Schema 可生成，并已导出到发布运维目录。
- Epic10 发布验收脚本 `backend/scripts/verify_epic10_release_ops.py` 可运行。
- Sprint 12 验收硬化脚本 `backend/scripts/verify_sprint12_acceptance.py` 可运行。

## 6. 退出标准

- P0 验收项全部通过，P1 验收项无阻塞发布的问题。
- 高风险操作审计日志可写入并可按资源、操作者和时间范围查询。
- `/api/v1/health` 与 `/api/v1/health/dependencies` 可用于基础探活和依赖配置检查。
- 备份恢复、发布检查和回滚说明已回填到运维文档。

## 7. 角色分工

- 产品：确认业务验收项与演示链路。
- 研发：执行后端编译、前端构建、OpenAPI 导出、Epic10 验证脚本和 Sprint 12 验收脚本。
- 测试：按验收清单回填结果，并记录遗留问题。
- 运维：确认测试环境配置、备份恢复和发布检查命令。
