# 03 系统设计

本目录用于承接需求冻结后的系统设计文档，是瀑布模式中的关键评审输出区。

建议文档关系如下：

- 总体设计说明书：说明系统全貌、模块划分、部署与主链路。
- 详细设计说明书：说明模块内部实现、关键流程与页面到接口映射。
- 接口设计说明：说明 API、请求响应、错误码、鉴权与约束。
- 数据模型设计：说明实体关系、表结构、索引与状态字段。
- 数据库设计：面向开发落地整理表结构、索引和数据字典。

建议先写顺序：

1. 总体设计说明书
2. 数据模型设计
3. 数据库设计
4. 接口设计说明
5. 详细设计说明书
