# RAG-Lab 项目导航

## 1. 文档目标

本文件用于统一管理本项目在“瀑布里程碑 + 敏捷迭代”模式下的文档入口、编写顺序与交付边界。

建议采用以下工作方式：

- 瀑布负责阶段关口：立项、需求冻结、总体设计评审、详细设计评审、测试验收、上线交付。
- 敏捷负责阶段内推进：需求拆分、Sprint 排期、开发联调、缺陷修复、持续验收。
- 文档既服务评审，也服务开发执行，避免只写“汇报型文档”。

## 4. 推荐编写顺序

1. 完成需求规格说明书和原型设计文档，并进行需求评审。
2. 形成项目计划与里程碑，明确阶段关口、角色分工与版本目标。
3. 编写总体设计说明书，明确模块边界、部署方式、对象关系和接口范围。
4. 编写详细设计说明书、接口设计说明、数据模型设计。
5. 建立产品待办清单和首个 Sprint 计划，把需求拆到迭代。
6. 编写测试计划与验收清单，贯穿开发、联调、上线前验收。

## 6. 建议阶段划分

建议把项目拆成以下 6 个阶段：

1. 需求与原型阶段
2. 总体设计阶段
3. 详细设计阶段
4. 开发与联调阶段
5. 测试与验收阶段
6. 发布与复盘阶段

每个阶段都走“阶段评审通过后进入下一阶段”，但阶段内开发采用 Sprint 推进。
